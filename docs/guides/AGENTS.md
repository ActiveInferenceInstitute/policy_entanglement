# AGENTS.md — `docs/guides/`

Workflow-oriented guides for human and agent contributors.

## Contents

| File | Audience | Purpose |
|---|---|---|
| `build_run.md` | Onboarding contributor | End-to-end: clone → `uv sync` → `lake build` → `pytest` → `run_all.py`. |
| `quickstart_recipes.md` | Returning contributor | Common task recipes: regenerate one figure, add a theorem, add a citation, swap a hyperparameter. |
| `styleguide.md` | Author of new prose / code | Manuscript token conventions, citation slug format, prose style, Lean naming. |
| `testing.md` | Author of new tests | TDD loop, no-mocks policy, coverage gate, marker conventions (`requires_pymdp`, `slow`). |

## Rules

- Guides are **task-oriented**, not topic-oriented. If a contributor has a
  goal (`I want to add a figure`), the guide should walk them from start to
  artifact.
- Every command shown in a guide must be runnable from the project root in
  the `uv`-managed virtualenv. Verified by the `scripts/run_all.py` smoke
  test for the canonical workflow.
- Style guidance lives here, not in `README.md`. Keep `README.md` to
  high-level orientation; defer to the appropriate guide.

## Adding a new guide

1. Pick a clear task verb in the title (`build_run`, `add_a_theorem`, …).
2. Open with a one-line problem statement, then the minimal recipe, then
   pitfalls / variants.
3. Cross-link from `[README.md](README.md)` and the parent
   [`../README.md`](../README.md) so it is discoverable.
