# Manuscript & Code Styleguide

This page is the **single contract** between the manuscript prose,
the figure / simulation pipeline, the Lean boundary fragment, and the
test suite.  Any violation of these rules is a CI failure: the gates
under [`scripts/validate_manuscript.py`](../../scripts/validate_manuscript.py)
and the test suite at [`tests/`](../../tests/) are the enforcement
mechanisms.

The styleguide is composed of six hard requirements.  Each has its
own page; this hub gives the headline rule and links to the details.

| § | Rule | Detail page |
|---|---|---|
| 1 | **Real, configurable, hyperparameter-injected simulations.**  Every grid size, seed, sweep range, rollout horizon, and observation used by a figure / sweep / validation script must come from [`src/simulation/hyperparameters.py`](../../src/simulation/hyperparameters.py).  No `np.linspace(0, 6, 121)` literal in a script — only `H.PARAMETER_SWEEP_LAMBDAS.values()`. | [`styleguide/hyperparameters.md`](styleguide/hyperparameters.md) |
| 2 | **Auto-injected manuscript variables.**  Every numeric value the manuscript prose mentions ($I(0.5)$, $\lambda^\star(0.9)$, grid counts, agreement tolerances, rollout step counts, …) must come from `output/data/manuscript_variables.json` via a `[[VAR:key]]` token.  No hardcoded numbers in `manuscript/*.md`. | [`styleguide/manuscript-variables.md`](styleguide/manuscript-variables.md) |
| 3 | **Auto-numbered, registry-resolved equations.**  Every display equation is auto-numbered as `S.K` (`S` = section number, `K` = within-section sequence).  Cross-references use `[[EQ:label]]` / `[[EQREF:label]]`. | [`styleguide/equations.md`](styleguide/equations.md) |
| 4 | **Auto-numbered, captioned figures.**  Every figure is registered in `manuscript/refs/labels.yaml` with a caption that names the exact generation method (function path + grid hyperparameter constants).  Figures are emitted via `[[FIG:label]]` and referenced via `[[FIGREF:label]]`. | [`styleguide/figures.md`](styleguide/figures.md) |
| 5 | **Resolved citations end-to-end.**  Every `[@key]` in `manuscript/*.md` resolves through `manuscript/refs/citations.yaml`; topic-grouped bibliographies are emitted via `[[CITELIST:topic]]`; the validator fails on any unresolved key. | [`styleguide/citations.md`](styleguide/citations.md) |
| 6 | **American English, evidence-first prose.**  Prose, captions, and docs use American spelling and make methods/results/discussion claims by naming artifacts, scripts, tests, and citations rather than relying on assertion alone. Historical code identifiers and original citation titles are left unchanged. | [`styleguide/prose.md`](styleguide/prose.md) |

Each detail page documents *how* its requirement is satisfied, what
the validator checks, and how to extend the system without breaking
it.

## CI / pre-commit gates

The full chain (run from the project root):

```bash
uv run python scripts/manuscript_variables.py     # populates output/data/manuscript_variables.json
uv run python scripts/inject_manuscript_variables.py  # renders manuscript with all tokens resolved
uv run python scripts/validate_manuscript.py      # gate: tokens, citations, ranges, hardcoded refs
uv run pytest tests/                              # gate: numerical contracts, figure smoke, registry
```

A green run means:

* every numeric value in prose flows from JSON, JSON flows from
  hyperparameters, sentinel tuples, and live registry scans recorded
  in `output/data/manuscript_variables.json`;
* every figure has been generated, registered, captioned with method
  attribution, and summarized in the live release-readiness report;
* every equation is auto-numbered and every cross-ref resolves
  through `manuscript/refs/labels.yaml`;
* every citation is registered and reachable through
  `manuscript/refs/citations.yaml`, with live counts generated by the
  manuscript-variable pipeline.
* prose-style files use American English, while code identifiers with
  historical spellings are treated as API names rather than prose.

If any of these fail, **the manuscript does not render in CI** —
this is by design.
