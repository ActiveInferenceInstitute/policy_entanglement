# AGENTS.md — `docs/reference/`

Generated and reference-style documentation. Anything in this directory is
either auto-generated by a script or is a stable reference index that does not
walk a contributor through a workflow (those go in `docs/guides/`).

*Latest generated audit:* live theorem/status counts come from the
generated theorem map and `manuscript/refs/labels.yaml`; live pytest,
PDF, figure, and artifact counts come from
`output/reports/release_readiness.json`,
`output/reports/test_results.json`, and `output/MANIFEST.md`.
Historical round deltas live in `docs/CHANGELOG.md` and
`docs/reference/veridical_status.md`.

## Rules

1. **Generated files are read-only.** `_theorem_map.md` is regenerated by
   `scripts/generate_theorem_map.py`. Edit the generator, not the file. The
   leading underscore signals that it is generated.
2. **Hand-authored references must mirror reality.** Every API table cell
   here must match the actual `src/<module>/<file>.py` symbol. Verified by
   the auto-generated test in
   [`tests/test_python_api_coverage.py`](../../tests/test_python_api_coverage.py).
3. **Do not walk a contributor through a task.** Workflow guides live in
   `docs/guides/`. References are catalogs, indexes, and lookup tables.
4. **Use stable anchors.** When a section is referenced from a manuscript
   chapter, the anchor must persist across rewrites; prefer
   `## My Heading {#stable_anchor}` style rather than implicit slug
   targeting.
