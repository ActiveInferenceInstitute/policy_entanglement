## Contracts

1. **Every** `[[FIG:label]]` / `[[FIGREF:label]]` / `[[EQ:label]]` / `[[EQREF:label]]` /
   `[[SECREF:label]]` / `[[THM:label]]` / `[[THMREF:label]]` / `[[LEAN:label]]` /
   `[@citekey]` token in any `manuscript/*.md` source file must resolve to an
   entry here. The validator fails the build otherwise.
2. **Every** entry here must be referenced from at least one source file â€”
   otherwise it is dead weight (carries cost, no benefit). Run the audit in
   [`tests/test_manuscript_registry.py`](../../tests/test_manuscript_registry.py)
   to detect drift.
3. **Numeric values** (display numbers `5.1`, `7.4`, etc.) live only here.
   Prose must use the registry-backed token; the renderer substitutes the
   display number at injection time. Do not hand-author `Theorem 5.1` in body
   text.
4. **Lean wiring** (`lean_module:` / `lean_name:`) is what powers the
   `[[LEAN:label]]` injection in `S05_lean_code_skeleton.md`. When you add a
   theorem with `status: proved | boundary | forwarder | witness`,
   the live Lean source is extracted at render time from
   `lean/ActinfPolicyEntanglement/<lean_module>.lean`.

## Allowed theorem statuses

| Status | Meaning |
|---|---|
| `proved` | Boundary proof exists for the registered boundary statement; row-level `faithfulness:` distinguishes substantive rows from statement-restricted rows. |
| `boundary` | Witness-form theorem â€” the boundary fragment certifies the algebraic skeleton; full analytic content is supplied by the caller (numerical Python layer or separate Mathlib4 discharge layer). |
| `forwarder` | Lean module re-exports a theorem from another module. |
| `witness` | Theorem is stated in witness-consuming form; the analytic content is delivered as a hypothesis. |
| `sketch` / `deferred` | Retired statuses. Do not use for new rows; every current numbered theorem has live Lean wiring. |

## Schema invariants

- `topic_order:` in `citations.yaml` is **the** ordering used by the
  auto-generated bibliography in `99_bibliography.md`. Adding a new topic
  requires adding it to both `topic_order:` (for ordering) and `topic_titles:`
  (for the section header).
- Section labels are dotted (`decomposition.proof`, `lambda_deformation.precision`)
  and may map to subsection headings or anchors inside a chapter file.
- Figure / equation labels are flat snake_case (`coupling_tax_quadratic`,
  `closed_form_F`).

## Adding a new entry

1. Edit `labels.yaml` or `citations.yaml`.
2. Reference the new entry from at least one `manuscript/*.md` file.
3. Re-run `uv run python scripts/inject_manuscript_variables.py`; the
   renderer will resolve the new token and the validator will confirm it
   appears in the rendered output.
4. For citation entries: re-run `scripts/run_all.py` so the bibliography
   regenerates and the auto-generated test
   [`tests/test_manuscript_registry.py`](../../tests/test_manuscript_registry.py)
   sees the new key.
